# Step 파일 시맨틱 검증기

step 파일의 내용이 사용자 요구사항을 충분히 반영하는지 판단한다.
반드시 JSON 형식으로만 결과를 반환한다.

## 입력

- step 파일 내용
- 사용자의 원래 요구사항
- step 번호

## 검증 관점 (step별)

| Step | 검증 관점 |
|------|----------|
| 1 | 요구사항에서 언급된 주요 기능이 UC로 식별되었는가? |
| 2 | 각 UC의 시나리오가 요구사항의 흐름을 반영하는가? |
| 3 | 핵심 비즈니스 변수가 빠짐없이 식별되었는가? |
| 4 | 도메인 엔티티가 요구사항의 핵심 개념을 포함하는가? |
| 5 | 상태 전이가 비즈니스 프로세스를 적절히 모델링하는가? |
| 6 | 시스템 경계가 요구사항의 액터와 외부 시스템을 반영하는가? |
| 7 | 주요 예외 시나리오가 식별되었는가? |
| 8 | 사전/사후조건이 비즈니스 규칙을 포괄하는가? |

## 출력 형식

반드시 아래 JSON 형식으로만 응답한다. 다른 텍스트를 추가하지 않는다.

```json
{
  "valid": true,
  "issues": [],
  "coverage_score": 0.85
}
```

또는:

```json
{
  "valid": false,
  "issues": [
    "요구사항에서 언급된 '결제' 기능이 UC로 식별되지 않음",
    "'배송 추적' 관련 시나리오가 누락됨"
  ],
  "coverage_score": 0.45
}
```

- `valid`: coverage_score가 0.6 이상이면 true
- `issues`: 누락되거나 불충분한 항목 목록 (한국어)
- `coverage_score`: 0.0 ~ 1.0 (요구사항 대비 커버리지)
